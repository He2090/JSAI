# JSAI 工作助手项目维护说明

本文档用于后续迭代、排查问题和交接维护。当前项目已经从早期 WebView/`index.html` 方向，迁移为以 WinForms 为主的桌面工作台，同时包含会员服务、自动更新、客户端配置工具和本地/云端模型调用链。

> BUG 修复、版本迭代和发包台账统一记录在 `docs/系统维护记录.md`。以后每修复一次 BUG，必须同步追加一条维护记录。


## 5. AI 漫剧节点流程

节点类型集中在：

- `F:\极速漫剧生成器\JSAI\WinApp\WorkflowModels.cs`

当前 AI 漫剧主流程：

```text
故事大纲 -> 故事剧本 -> 创意描述 -> 分镜图片 -> 分镜图拆解 -> 分镜视频 -> 视频合集
                  ^
故事大纲 -> 角色设计 -> 分镜图片
```

连线规则约定：

- `故事大纲` 只能连接 `故事剧本`、`角色设计`。
- `故事剧本` 只能连接 `创意描述`。
- `角色设计` 只能连接 `分镜图片`。
- `创意描述` 只能连接 `分镜图片`。
- `分镜图片` 只能连接 `分镜图拆解`。
- `分镜图拆解` 只能连接 `分镜视频`。
- `分镜视频` 只能连接 `视频合集`。

这些规则应在 `WorkflowNodeCatalog.CanConnect(...)` 或相邻逻辑中维护。后续新增节点时，要同步更新：

- 节点目录列表。
- 节点连接规则。
- 节点默认模型类型。
- 右侧模型选择框。
- 保存/载入迁移逻辑。

### 5.1 故事大纲

功能：

- 输入故事简介。
- 选择类型、背景、视觉风格、集数、单集时长。
- 调用文本模型生成结构化大纲。

维护点：

- 输出格式必须稳定，方便后续 `故事剧本`、`角色设计` 解析。
- 大纲重置后要触发下游角色列表刷新，避免旧角色残留。

### 5.2 故事剧本

功能：

- 从上游大纲解析章节。
- 选择章节与拆分集数。
- 生成分集创意描述。
- 自动在右侧生成 `创意描述` 子节点并连线。

维护点：

- 大纲格式变化时，必须同步更新章节解析器。
- 重新生成剧本时，应清理旧创意描述节点，避免版本混乱。

### 5.3 创意描述

功能：

- 显示单集创意描述。
- 点击 `拆分为影视分镜` 后，在同一个节点内显示文字分镜列表。
- 每条分镜可编辑。

维护点：

- 创意描述节点不显示预览图。
- 分镜图预览应该属于 `分镜图拆解` 节点。
- 重新拆分时不要丢失节点输出可见性。

### 5.4 角色设计

功能：

- 从故事大纲读取角色清单。
- 每个角色先生成角色档案。
- 档案完成后生成九宫格。
- 九宫格完成后生成三视图。
- 支持详情窗查看角色说明、提示词、九宫格、三视图。
- 支持角色资产保存。

角色资产保存目录：

```text
XMwenjian\ai漫剧\<项目名>\jiaose\<角色名>\
```

维护点：

- 九宫格/三视图的本地 ComfyUI 生成，优先走正脸参考图，再用参考图生成其它图。
- 云端图片生成不应提交反向提示词，除非目标云端 API 明确支持。
- 男性角色容易漂成女性时，要同时在正向和反向提示词里约束性别。

### 5.5 分镜图片

功能：

- 从 `创意描述` 和 `角色设计` 读取分镜与角色信息。
- 生成九宫格或六宫格分镜页。
- 每个九宫格作为一页，可翻页、放大查看、编辑描述。

关键维护点：

- 实际 prompt 必须强制吃 `AppearancePrompt + CostumeNotes`。
- 参考图优先级：`三视图` > `九宫格` > `正脸参考图`。
- 本地 ComfyUI 参考图 workflow 不要直接喂整张三视图/九宫格设定板，否则模型会复读设定板。
- 分镜图禁止生成文字、字幕、SFX、UI 文案、水印。相关约束应同时进入正向和反向 prompt。
- 本地模型质量不足时，分镜图片建议默认云端生成，本地作为备用。

### 5.6 分镜图拆解

功能：

- 从 `分镜图片` 的分页图里裁切单个镜头。
- 每个分镜显示图片在上、文字在下。
- 每个分镜支持编辑。
- `编辑` 前新增 `重新获取`，可从原始分镜图片页里单独重新裁切当前镜头。

维护点：

- 不要重新跑整页生成；`重新获取` 只处理单个分镜的裁切。
- 如果新增节点导致拆解图消失，要检查 `StoryboardShots[*].SplitImagePath` 是否仍然存在并被 UI 使用。

### 5.7 分镜视频

功能：

- 从 `分镜图拆解` 获取镜头列表。
- 勾选需要生成视频的分镜。
- 生成融合参考图。
- 生成视频提示词。
- 根据本地/云端视频模型生成视频。
- 云端成功后下载视频，并生成 `视频预览` 节点。
- 连接到 `视频合集` 后，生成的视频要自动进入合集可选列表。

关键维护点：

- 文本提示词模型默认优先本地文本模型；没有本地模型才退云端，除非用户在节点里手动指定。
- 云雾视频查询真实可用地址可能在 `detail.video_url`，不要只读顶层 `url`。
- 云雾任务 ID 形如 `veo3.1-fast:...`，不能当 URL 使用。
- `按ID获取` 应只把 `http/https` 当查询地址，其它都当任务 ID。
- 如果云雾后台状态成功但节点未回填，优先排查任务查询结构、视频下载和 `视频预览 -> 视频合集` 刷新链。

### 5.8 视频合集

功能：

- 从上游 `视频预览` 或 `分镜视频` 读取视频片段。
- 支持本地视频导入、素材移出、小窗预览和按时间线顺序合成。
- 时间线包含视频、文字、图片、字幕、音频五条基础轨道。
- 预览窗口可显示文字/图片叠加层，叠加层位置会写入节点参数并参与最终合成。

当前维护点：

- 成功回收的视频要加入 `视频合集` 的片段列表。
- 手动导入的视频要加入 `VideoCollectionImportedAssets`，并同步 `VideoCollectionSelectedArtifactPaths` 与 `VideoCollectionTimelineClips`。
- 文字/图片叠加层要保存到 `VideoCollectionOverlayItems`；文字必须经过简体中文过滤，图片路径需要随 `.myai` 项目包复制。
- 合成前应明确所选片段顺序。
- 无片段或未选中片段时，预览区要显示空剪辑初始化画布和时间码，不能退回纯黑框。
- FFmpeg 合成链路需要覆盖视频合并、字幕、外部音轨、文字叠加、图片叠加和转场；如果外部工具失败，必须记录依赖位置和失败日志。

### 6.1 文生图

功能：

- 用户输入提示词。
- 文本模型先优化正向/反向提示词。
- 图片模型生成图片。
- 支持普通、九宫格、三视图模式。
- 支持本地优先、云端回退。
- 支持图片预览、打开文件、打开目录、图片保存、清空。

注意：

- 本地图片模型可以使用反向提示词。
- 云端图片模型默认不要提交反向提示词，除非 API 明确支持。
- 动物/性别等主体必须把用户原始提示词并回正向 prompt，避免提示词模型重写跑偏。

### 6.2 文生视频

功能：

- 用户输入文字描述。
- 文本模型优化视频提示词。
- 视频模型生成视频。
- 参数应使用视频参数：画幅、时长、画质。

注意：

- 本地视频模型通常只有模型名称，没有云端那种主模型/子模型层级。
- 云端模型可配置视频接口、视频大模型、子模型。

### 6.3 文图生视频

功能：

- 用户选择一张或多张参考图。
- 输入视频提示词。
- 文本模型优化提示词。
- 视频模型生成视频。

注意：

- 本地 ComfyUI 图生视频需要使用图生视频 workflow。
- 云端模型按目标 API 的图片字段格式提交，不要混用 `data:image/...` 和纯 base64。

## 7. 模型设置与路由

模型配置入口：

- `ModelSettingsForm.cs`
- `AddModelForm.cs`
- `ModelConfig.cs`

模型类别：

- `Text`
- `Image`
- `Video`

模型来源：

- `Local`
- `Cloud`

基本原则：

- 模型设置里填写的是服务地址、模型 ID、Key、类别和来源。
- 本地 ComfyUI 图片模型允许额外配置 `工作流 JSON`。这个字段只对 ComfyUI 图片模型显示和生效；运行时优先加载选中的 JSON，并替换 `{{positive_prompt}}`、`{{negative_prompt}}`、`{{input_image}}`、`{{seed}}`、`{{width}}`、`{{height}}`、`{{denoise}}`、`{{filename_prefix}}`、`{{model_id}}` 等占位符。未使用占位符的旧工作流继续按标准 ComfyUI 节点兼容注入，不再把 checkpoint 匹配作为硬性测试条件。
- Key 使用 DPAPI 加密保存，配置中可能显示为 `dpapi:` 前缀。

文本模型默认策略：

1. 节点手动选择的模型优先。
2. 没有手动选择时，优先使用本地文本模型。
3. 本地文本模型不存在或不可用时，再使用云端文本模型。

图片模型默认策略：

1. 节点手动选择优先。
2. 默认优先本地图片模型。
3. 本地不可用时询问是否切换云端模型。

视频模型默认策略：

1. 节点手动选择优先。
2. 本地视频模型可用时优先本地。
3. 云端视频模型用于 Grok、Veo、Sora 等任务。

中转 API：

- 云雾等中转 API 配置应放在中转 API 设置里，不要在每个模型里重复维护 Key。
- 视频节点可以有视频接口、视频大模型、子模型三层。
- Grok 示例中子模型曾收敛为 `grok-video-3-10s`，实际提交时可能需要映射为 API 支持的真实模型 ID。

## 8. ComfyUI Workflowsapi

本地 ComfyUI workflow 模板目录：

```text
F:\极速漫剧生成器\JSAI\WinApp\bin\Debug\net8.0-windows\Workflowsapi
```

当前已知文件：

- `Get_imageAI.json`：基础文生图模板。
- `文生图API1.0.json`：文生图模板，偏正式出图。
- `FaceimgtoimgWorkflow.json`：图生图模板，适合参考图生成图片。
- `LTX2IMGTOVDO.json`：图生视频模板。
- `ImgandTextTovideo1.0.json`：图文生视频模板，适合文图生视频调用。

维护原则：

- 模型设置中填写 ComfyUI 地址，例如 `http://127.0.0.1:8188/`，类别选 `Image` 或 `Video`。
- workflow JSON 由程序按节点类型自动选择。
- 修改 workflow 后，应检查 `WorkflowRuntimeService.cs` 里的节点 ID 注入逻辑是否仍然匹配，例如正向 prompt、反向 prompt、宽高、seed、LoadImage 输入等。
- LTX 图生视频 workflow 中的尺寸节点必须逐级核对，尤其是 `ImageScaleBy.scale_by`。如果该值小于 `1.0`，即使前面设置了 720P/1280 宽高，最终也会被中间节点降采样成低清视频。
- 如果 workflow 中节点 ID 发生变化，必须同步更新注入代码，否则看似调用成功但参数不会生效。

当前 ComfyUI 调用维护点：

- 文生图：优先使用文生图模板。
- 图生图：使用 `FaceimgtoimgWorkflow.json`。
- 文图生视频：应接入 `ImgandTextTovideo1.0.json`。
- AI 漫剧分镜视频本地方案可使用图生视频模板，但需要确认输入图、提示词和视频输出节点。

## 9. 云雾 API 与视频回收

云雾视频任务常见问题：

- 后台显示成功，但客户端没有拿到视频。
- 任务 ID 带冒号，例如 `veo3.1-fast:...`，容易被误判成 URI scheme。
- 查询接口可能返回多个 URL 字段，其中真正可用的可能是 `detail.video_url`。
- 有些接口返回顶层 `url`，但该 URL 可能不可下载。

维护建议：

- 创建任务时记录任务 ID。
- 成功、失败都要写入模型调用日志。
- `按ID获取` 应允许用户粘贴任务 ID 后重新查询。
- 查询候选字段应包含：`detail.video_url`、`video_url`、`download_url`、`play_url`、`url`。
- 下载时逐个候选 URL 尝试，直到拿到真正的 `video/mp4` 或可保存的视频二进制。
- 下载成功后要同步：
  - 节点输出。
  - `视频预览` 节点。
  - 已连接的 `视频合集`。

模型调用日志：

- 成功日志：`logs\model-call-success.log`
- 失败日志：`logs\model-call-failure.log`

AI 漫剧内建议继续按内容拆分：

```text
XMwenjian\ai漫剧\<项目名>\dagang\
XMwenjian\ai漫剧\<项目名>\juji\
XMwenjian\ai漫剧\<项目名>\jiaose\<角色名>\
XMwenjian\ai漫剧\<项目名>\fenjing\
XMwenjian\ai漫剧\<项目名>\fenjingRH\
```

维护点：

- 保存项目时要尽量把大纲、剧集、角色、分镜、视频等文件跟 `.myai` 项目放在一起。
- 角色说明和提示词用 `.txt` 保存。
- 分镜描述也应保存 `.txt`。
- 视频片段和合集应记录来源节点、任务 ID、生成模型、提示词和本地文件路径。

## 12. 常见排障

### 12.1 编译失败，提示文件被占用

通常是 `WinApp.exe` 或 `UpdateServer.exe` 正在运行。关闭后重新执行：

```powershell
dotnet build F:\极速漫剧生成器\JSAI\JSAI.sln
```

### 12.2 后台打开是一串 JSON

打开了接口地址，例如：

```text
http://127.0.0.1:5157/api/admin/overview
```

正确后台页面是：

```text
http://127.0.0.1:5157/admin
```

### 12.3 云雾视频后台成功但客户端没有视频

优先检查：

- 本地失败日志。
- 任务 ID 是否保存到节点。
- 是否把 `veo3.1-fast:...` 误当 URL。
- 查询结果是否有 `detail.video_url`。
- 下载成功后是否触发 `视频预览` 和 `视频合集` 刷新。

### 12.4 ComfyUI 报 `response ended prematurely`

通常是本地 ComfyUI 下载图片时连接提前断流。运行时已加重试和流式读取，但仍可能由 ComfyUI 服务本身、显存压力或输出文件未稳定导致。可尝试：

- 重启 ComfyUI。
- 降低生成尺寸。
- 先确认 `http://127.0.0.1:8188/object_info` 可访问。
- 查看 ComfyUI 控制台是否报错。

### 12.5 本地文本模型 OpenAI 接口看似失败

有些本地 OpenAI 兼容服务会把内容放在 `reasoning_content`，而不是标准 `choices[0].message.content`。解析时要兼容。

模型 ID 必须使用服务返回的完整 ID，例如：

```text
google/gemma-4-26b-a4b
```

不要只填短名：

```text
gemma-4-26b-a4b
```

### 12.6 分镜视频出现伪文字、竖条或乱码块

本地视频模型不适合直接生成字幕、招牌、屏幕 UI、海报、标签等文字。即使提示词要求“仅简体中文”，模型也可能把参考图中的牌匾或小字放大成伪文字、闪烁竖条或乱码块。

维护规则：

- 分镜视频执行提示词必须禁止画面内文字。
- 视频模型 prompt 的文字禁令必须短而集中，不要反复堆叠 `subtitle/caption/text` 及其变体；正向提示词只保留一条“画面内无可见文字”的规则，负面提示词只保留关键项。
- 对白、字幕和文字内容只作为后期字幕/剪辑信息，不交给视频模型直接绘制。
- 后期字幕写入视频前必须做简体中文过滤，删除英文、数字、路径、模型名、乱码和不可读符号；过滤后为空则不生成该字幕行。
- 如果参考图本身带有牌匾、字幕、标签或不可读文字，视频提示词要要求忽略/移除这些区域，保持为空白自然表面。
- ComfyUI 视频工作流的负面提示词保持精简，优先包含 `subtitles`、`captions`、`visible text`、`watermark`、`logo`、`text artifacts`、`fake subtitles` 等关键约束，避免长负面提示词挤占模型注意力。
- 生成前如果分镜图片已经有明显伪文字，优先重新生成干净分镜图，再做视频。
- 分镜视频提示词结构必须保留“风格、全局规则、角色一致性、时长、场景、出场角色、景别、机位、镜头运动、画面、光影、视觉效果、声音、对白/字幕”等字段。不要把缺失的运镜字段直接默认成“固定”；应交给文本模型按画面、情绪和景别推导具体运镜。
- `声音` 字段在发送给本地视频模型时必须转成明确英文 SFX；不能降级成 `subtle production cue` 这类泛化提示。未明确要求音乐时，应禁止无关背景音乐。
- LTX/ComfyUI 图生视频 workflow 中如果存在 `TextGenerateLTX2Prompt` 节点，必须同步写入 APP 生成的英文执行提示词，避免空 prompt 或模板提示影响画面和声音。
- 提交 LTX 图生视频 workflow 前，运行时代码应拦截 `ImageScaleBy.scale_by < 1.0` 的降采样设置；workflow 模板默认值也应保持 `1.0`，避免输出片段低于目标清晰度。

### 12.7 分镜图片分辨率与可见文字规则

分镜图片节点按单镜头 1080P 生成：

- 横屏分镜单格：`1920x1080`。
- 竖屏分镜单格：`1080x1920`。
- 合成分镜页时每个格子也保持对应 1080P 尺寸，保证后续“分镜图拆解”拿到的镜头图不再被压小。
- 分镜视频必须使用高清分镜参考图；如果旧项目里的参考图低于目标尺寸，提交视频模型前先生成 `_高清参考_` PNG，再用这张图做图生视频参考。

分镜图片允许画面内出现文字，但只允许准确简体中文字符和中文标点。提示词与负面提示词必须禁止英文、拼音、阿拉伯数字、繁体、日韩文字、随机符号、伪文字和乱码。本地叠加字幕也要在写入图片前过滤，只保留中文字符和中文标点。

### 12.8 生成任务处理中提示框

所有会调用文本、图片、视频模型或 ComfyUI 工作流的入口，都必须通过 `BeginProcessingScope(...)` 打开“任务处理中，请稍后”的心跳浮窗，结束或失败后由作用域自动关闭。

维护规则：

- 顶部“执行工作流”、右侧“运行当前节点”、画布节点快捷运行，都必须覆盖。
- 节点动作按钮如生成分镜、生成视频、按任务 ID 回收视频、生成合集等，都必须覆盖。
- 角色节点的生成档案、九宫格、三视图等快捷动作，也必须覆盖。
- 如果一个入口内部还会调用 `RunNodeAsync(...)`，允许嵌套作用域，统一复用当前弹窗并更新详情文字，不能重复弹多个窗口。
- `TaskProcessingForm` 的中文文案必须保持 UTF-8，避免窗口标题或正文出现乱码。

### 12.9 视频合集内嵌预览

视频合集节点的小窗内嵌预览不能依赖 Windows MCI `mpegvideo` 播放 MP4。ComfyUI/LTX 常见输出为 H.264/AAC MP4，MCI 对这类文件兼容性不稳定，容易触发“内嵌预览不可用”。

维护规则：

- 小窗内嵌预览使用 WPF `MediaElement` 并通过 `ElementHost` 嵌入 WinForms 面板。
- 播放、暂停、停止、回到开头按钮应优先控制内嵌播放器。
- 如果内嵌播放器失败，保留“放大”或系统播放器作为兜底，不要误判为素材丢失。
- `WinApp.csproj` 必须同时启用 `UseWindowsForms` 和 `UseWPF`。

### 12.10 角色名主键与模型返回名

角色设计节点中的 `CharacterDesignEntry.Name` 是资产写回主键，不能被文本模型返回的 `name` 覆盖。大纲中的角色名可能包含身份补充，例如“苏家主（苏父）”“林越的岳母（王氏）”；模型常会简化成“苏家主”“王氏”。如果覆盖主键，九宫格或三视图生成成功后，刷新角色列表会按原始大纲角色名重建条目，导致生成结果挂到临时简名上并被丢掉。

维护规则：

- 角色档案解析时，模型返回的 `name` 只能作为别名保存，不能覆盖已有 `entry.Name`。
- 只有当 `entry.Name` 原本为空时，才允许使用模型返回名填充。
- 资产路径写回、状态写回、保存项目和导出资源都必须使用原始角色名匹配。

## 13. 维护建议与技术债

优先整理项：

- 把 `Workflowsapi` 的 workflow 与节点路由做成可配置映射，而不是继续在代码里硬编码文件名。
- 把云雾视频 `create/query/download` 的真实返回结构做成单独适配器，避免散落在运行时大文件里。
- 将 `WorkflowRuntimeService.cs` 继续拆分为文本、图片、视频、ComfyUI、云雾、项目产物六类服务。
- 给 `.myai` 增加版本字段和迁移器，减少节点参数变更后的兼容问题。
- 视频合集已具备基础轨道式编辑器；后续可继续补齐片段裁剪、轨道缩放、字幕逐条编辑、音频波形和导出预设。
- 分镜图片本地 ComfyUI 质量不稳定，建议云端优先，本地作为备用；如果要提升本地质量，需要 IPAdapter、ControlNet 或更强分镜构图 workflow。
- 服务端后台继续补充管理员按人筛选、操作日志导出、财务报表导出。

新增节点时必须检查：

- `WorkflowNodeCatalog` 类型与显示名称。
- `WorkflowNodeParameters` 参数与默认值。
- `WorkflowNodeCard` UI。
- `WorkflowRuntimeService` 执行逻辑。
- `WorkflowExecutor` prompt 与解析。
- `WorkflowStore` 保存/载入兼容。
- `ProjectLibraryExportService` 产物导出。
- 右侧模型选择框需要哪些模型类别。
- 是否需要连线规则。

## 14. 当前维护原则

- 先保证可用，再做 UI 精修。
- 所有模型调用必须有成功/失败日志。
- 云端任务必须记录任务 ID，便于失败后回收。
- 节点输出必须能保存到 `.myai` 和 `XMwenjian`。
- 本地模型默认优先，但失败时要给用户明确选择是否切云端。
- 涉及会员、后台、更新的代码要优先考虑安全边界，不允许未认证访问后台管理操作。
